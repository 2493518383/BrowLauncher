# BrowLauncher 产品说明书

## 产品简介

**BrowLauncher** 是一款浏览器快速启动器插件，让您可以在浏览器中一键启动本地应用程序和打开文件夹，提高工作效率。

- **作者**：花雨sec
- **微信**：byzy1010
- **版本**：1.0.0

## 功能特性

- 📁 **分类管理**：自定义应用分类，清晰组织您的工具
- 🚀 **快速启动**：一键启动本地程序或打开文件夹
- ✏️ **灵活编辑**：支持添加、编辑、删除应用和分类
- 💾 **数据持久化**：配置自动保存到浏览器本地存储
- 🎨 **美观界面**：简洁现代的 UI 设计
- 🌐 **多浏览器支持**：支持 Chrome、Edge、Firefox

## 安装指南

### 环境要求

- **操作系统**：Windows 10/11
- **浏览器**：Chrome 88+、Edge 88+ 或 Firefox 91+
- **Python**：3.8+（需添加到系统 PATH）

### 第一步：安装浏览器插件

#### Chrome / Edge 浏览器

1. 打开浏览器，进入扩展管理页面：
   - Chrome：地址栏输入 `chrome://extensions/`
   - Edge：地址栏输入 `edge://extensions/`

2. 开启右上角的 **「开发者模式」**

3. 点击 **「加载已解压的扩展程序」**

4. 选择文件夹：`E:\BrowLauncher\extension`

5. 安装成功后，**记录显示的扩展 ID**（32位字母字符串）

#### Firefox 浏览器

1. 地址栏输入 `about:debugging#/runtime/this-firefox`

2. 点击 **「临时载入附加组件」**

3. 选择文件：`E:\BrowLauncher\extension-firefox\manifest.json`

### 第二步：安装本地消息服务

1. 进入目录 `E:\BrowLauncher\native-host\`

2. **双击运行** `install.bat`

3. 按提示选择浏览器类型：
   - 输入 `1`：仅安装 Chrome/Edge
   - 输入 `2`：仅安装 Firefox
   - 输入 `3`：全部安装

4. 输入第一步记录的 **扩展 ID**

5. 等待安装完成

6. **重启浏览器**

## 使用说明

### 打开插件

点击浏览器工具栏中的 **BrowLauncher 图标**，弹出主界面。

### 基本操作

| 操作 | 说明 |
|------|------|
| **点击应用** | 启动程序或打开文件夹 |
| **点击分类** | 切换到该分类 |
| **+📁 按钮** | 添加新分类 |
| **+📱 按钮** | 添加新应用 |
| **⚙️ 按钮** | 重置为默认数据 |
| **✏️ 按钮** | 编辑分类或应用（鼠标悬停时显示） |
| **关于作者** | 查看作者信息 |

### 添加应用

1. 点击右上角 **+📱** 按钮

2. 填写应用信息：
   - **应用名称**：显示名称（如：记事本）
   - **应用路径**：程序路径或文件夹路径
   - **启动参数**：可选，命令行参数
   - **所属分类**：选择分类

3. 点击 **保存**

### 路径示例

| 类型 | 路径示例 |
|------|----------|
| 系统程序 | `notepad.exe`、`calc.exe`、`cmd.exe` |
| 完整路径 | `C:\Program Files\Google\Chrome\Application\chrome.exe` |
| 文件夹 | `E:\work\project\`（会用资源管理器打开） |
| 带参数 | 路径：`chrome.exe`，参数：`--incognito` |

## 常见问题

### Q1：点击应用后提示 "Error when communicating with the native messaging host"

**原因**：本地消息服务未安装或配置错误

**解决方案**：
1. 确保已运行 `install.bat` 安装本地服务
2. 确保系统已安装 Python 并添加到 PATH
3. 重启浏览器后再试
4. 检查扩展 ID 是否正确

### Q2：重新安装插件后无法使用

**原因**：扩展 ID 变更导致配置失效

**解决方案**：
1. 记录新的扩展 ID
2. 重新运行 `install.bat`
3. 输入新的扩展 ID
4. 重启浏览器

### Q3：如何备份我的配置？

插件数据存储在浏览器本地存储中。目前暂不支持导出功能，建议截图记录重要配置。

### Q4：如何卸载？

1. 在浏览器扩展页面移除 BrowLauncher 插件
2. 运行 `E:\BrowLauncher\native-host\uninstall.bat` 清理注册表

## 技术原理

```
┌──────────────┐     Native Messaging     ┌─────────────┐     ┌──────────────┐
│  浏览器插件   │  ←─────────────────────→  │  Python Host │ ──→ │  启动程序    │
│  (popup.js)  │         JSON 通信         │  (host.py)  │     │  打开文件夹  │
└──────────────┘                          └─────────────┘     └──────────────┘
```

浏览器出于安全限制，插件无法直接执行本地程序。BrowLauncher 通过 Chrome/Firefox 的 **Native Messaging API** 与本地 Python 脚本通信，由 Python 脚本执行实际的程序启动操作。
